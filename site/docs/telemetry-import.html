Telemetry import

On any event page, Import video / telemetry… turns recordings into sessions with laps. Parsing happens entirely in the browser — for videos, via byte-range reads of the embedded telemetry track (a few MB of a multi-GB file). Files never leave your computer.

Supported sources

SourceWhat's readLap timing
Corvette PDR (Cosworth) MP4 The ctbx/marl telemetry track, read by byte range Beacon events + odometer recovery, or latitude/odometer lap recovery when a recording has no beacons (see below)
GoPro MP4 (Hero 5+) The GPS trace from the GPMF metadata track Start/finish line picker
Racelogic VBO
(VBOX; RaceChrono / TrackAddict / Harry's LapTimer exports)		 The GPS samples in the file The file's [laptiming] start line when present, otherwise the line picker

The start/finish line picker

GPS-only sources have no lap markers, so the import preview shows the track map you actually drove: click where the start/finish line is and laps are timed on each pass across it. Crossings are interpolated between 10–18 Hz GPS fixes, so times are accurate to roughly ±0.1–0.3 s and shown with a ~ prefix.

One picked line applies to every file in the batch, so a day's worth of sessions needs a single click. Racelogic's west-positive longitude convention is detected and mirrored automatically.

How PDR lap times are derived

The PDR logic is reverse-engineered from the recorder's telemetry track and validated against Cosworth Toolbox lap times:

  • PDR Beacon events mark start/finish crossings to the millisecond, but the recorder drops some crossings.
  • The cumulative odometer channel recovers the missing ones: beacon-to-beacon distance ÷ crossing count gives the lap length, and a missing crossing is the moment distance passes D0 + k × lapLength (accurate to ~0.05–0.3 s, shown with ~).
  • Crossings beyond the first/last beacon are extrapolated the same way and sanity-checked against GPS latitude.
  • The recording's GPS trace is decoded from the telemetry stream too, so a PDR file with no beacons gets the same track map and start/finish line picker as the other GPS sources. The import preview also reads the car channels and shows each session's top speed, max RPM and max lateral G, saved into the session notes.
  • Each imported session also stores per-lap channel data — speed for every source, plus RPM and lateral G for PDR — sampled along the lap's driven distance. On the event page the session's lap chips double as the picker for the expandable channel graphs: laps overlaid on a shared distance axis, comparing up to three laps side by side. (This is derived at import time; the recording itself is never uploaded, so re-import a file if you delete a session.)
  • If a recording's GPS can't be decoded, a beacon-less file still gets lap times: latitude as a function of odometer distance repeats every lap, so the lap length is that profile's autocorrelation peak and laps are cut every lap-length of distance. Start/finish alignment comes from a beacon-timed recording of the same track imported in the same batch; without one, laps are still full laps of the track, just cut from where the car first reaches pace rather than at the official line.
Millisecond-exact times are shown as-is; anything derived by interpolation or odometer recovery carries a ~ so you always know which laps are measured and which are estimated.
